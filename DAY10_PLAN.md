## Sprint Retrospective (Self) — 20-30 minutes

### What Went Well?
- [ ] List 3-5 things that went well
- [ ] Examples:
  - Database design was solid
  - No major technical blockers
  - Tkinter was easier than expected
  - Good progress each day
- [ ] Write in daily log or retrospective doc

### What Could Be Improved?
- [ ] List 2-3 things that could be better
- [ ] Examples:
  - Took longer than expected on [X]
  - Need better error handling in [Y]
  - Should have tested earlier
- [ ] Be honest but constructive

### What Will I Do Differently in Sprint 2?
- [ ] Pick 1-2 specific improvements
- [ ] Make them actionable
- [ ] Examples:
  - "Test each feature immediately after building"
  - "Take more frequent breaks"
  - "Commit code more often (2-3x per day)"

### Velocity Analysis
- [ ] Planned: 20 points
- [ ] Actual: ___ points
- [ ] If completed all 20: velocity = 20 points/sprint
- [ ] If completed less: adjust Sprint 2 accordingly
- [ ] If completed all 20 early: could add more to Sprint 2

### Sprint 2 Capacity Planning
- [ ] Sprint 2 has 14 points planned
- [ ] Based on Sprint 1 velocity, is this realistic?
- [ ] Sprint 2 is lighter, which is good (timeline is complex)
- [ ] Feel confident about Sprint 2 capacity

### Action Items for Next Sprint
- [ ] List 1-2 specific improvements to implement
- [ ] Add to Trello or notes

### Expected Deliverables:
- Retrospective notes
- Action items for Sprint 2
- Adjusted expectations if needed
